چون به Lovable Cloud دسترسی ندارید (به‌خاطر اینترنت ملی)، راه‌حل: یک پروکسی PHP کوچک روی هاست خودتان قرار می‌دهیم. سایت به جای اینکه مستقیم به `api.tarkhiskala.info` درخواست بزند، به `https://tarkhisun.com/api/hscode-search.php` درخواست می‌زند. چون این آدرس هم‌دامنه (same-origin) است، CORS کلاً منتفی می‌شود. خود فایل PHP از سمت سرور هاست شما (که محدودیت ندارد) به API اصلی وصل می‌شود.

## آنچه در پروژه تغییر می‌کند

1. ساخت فایل `public/api/hscode-search.php`
   - یک پروکسی ساده با cURL
   - اعتبارسنجی ورودی: `phrase` حداقل ۲ کاراکتر، نرمال‌سازی ارقام فارسی/عربی، `offset` و `limit` در محدوده مجاز
   - تنظیم هدرهای CORS و کش ۶۰ ثانیه
   - مدیریت خطا (502 در صورت خطای upstream)

2. به‌روزرسانی `src/lib/hscode-api.ts`
   - حذف فراخوانی Supabase
   - فراخوانی به `/api/hscode-search.php` روی همان دامنه
   - یک fallback مستقیم به API اصلی فقط برای محیط dev (لوکال)

3. (اختیاری) حذف پوشه `supabase/functions/hscode-search` که در پاسخ قبلی ساخته شده بود.

## مراحلی که شما باید انجام دهید (مرحله به مرحله)

1) پیش‌نیاز هاست
   - مطمئن شوید هاست شما PHP دارد (تقریباً همه هاست‌های اشتراکی دارند) و افزونه `cURL` فعال است.
   - اگر هاست فعلی فقط Static است (مثلاً GitHub Pages یا Netlify رایگان)، باید یک هاست با PHP استفاده کنید یا بگویید تا راه جایگزین (مثل Cloudflare Worker) را پیشنهاد بدهم.

2) دریافت بیلد جدید از Lovable
   - بعد از اینکه این پلن را تأیید کنید، تغییرات را اعمال می‌کنم.
   - در Lovable روی Publish/Build بزنید و فایل‌های جدید `dist` را دانلود کنید.

3) آپلود روی هاست خودتان
   - محتوای پوشه `dist` را مثل قبل در ریشه دامنه آپلود کنید.
   - مطمئن شوید فایل `dist/api/hscode-search.php` هم آپلود می‌شود (این فایل از مسیر `public/api/` به صورت خودکار وارد `dist/api/` می‌شود).

4) تست سریع روی دامنه
   - این آدرس را در مرورگر باز کنید:
     `https://tarkhisun.com/api/hscode-search.php?phrase=اسباب&limit=5`
   - باید JSON برگردد. اگر برگشت یعنی پروکسی کار می‌کند.

5) تست در صفحه `/hscode`
   - وارد `https://tarkhisun.com/hscode` شوید و کلمه‌ای جستجو کنید.
   - باید نتایج بدون خطای CORS نمایش داده شوند.

## اگر مشکلی پیش آمد

- اگر مرورگر روی فایل PHP خطای 404 داد: یعنی فایل در مسیر درست آپلود نشده. مسیر باید `/public_html/api/hscode-search.php` باشد.
- اگر JSON برنگشت و کد PHP خام نشان داده شد: PHP روی هاست فعال نیست؛ باید از پشتیبانی هاست بخواهید فعال کنند.
- اگر 502 داد: یعنی هاست شما به API دسترسی ندارد (مشکل از خود هاست با اپ‌استریم).

تأیید کنید تا تغییرات را روی پروژه اعمال کنم.
