ریشه مشکل پیدا شد: عدد ۰ از دو API می‌آید:

1. موجودی صندوق، کل ورودی، کل خروجی و سود نقدی از:
`/api/admin/treasury-summary.php`

2. دفتر کل صندوق از:
`/api/admin/treasury-ledger.php`

در دیتابیس ارسالی شما رکورد درست وجود دارد:
- `ts_card_payments`: پرداخت کاربر `200,000,000` با `status='confirmed'` و `to_treasury=1`
- `ts_treasury_ledger`: ردیف ورودی `200,000,000`
- پرداخت ادمین هم `rejected` است و نباید از صندوق کم شود

پس از نظر دیتابیس، خروجی درست باید این باشد:
- کل ورودی: `200,000,000 تومان`
- کل خروجی: `0 تومان`
- موجودی صندوق: `200,000,000 تومان`
- دفتر کل: یک ردیف ورود برای حسین مومنی

مشکل احتمالی فعلی این است که API در هاست خطا می‌خورد ولی فرانت‌اند خطا را فقط toast می‌کند یا صفحه را در حالت صفر/خالی نشان می‌دهد. ضمن اینکه `treasury-summary.php` هنوز به جدول ledger وابسته است و اگر query یا backfill روی هاست خطا بدهد، نتیجه قابل‌فهمی به کاربر نشان داده نمی‌شود.

برنامه اجرا:

1. `treasury-summary.php` را مقاوم می‌کنم تا موجودی را مستقیم از منبع اصلی حساب کند:
   - ورودی = مجموع `ts_card_payments.amount_irt` برای `status='confirmed'` و `to_treasury=1`
   - خروجی = مجموع `ts_card_admin_payments.amount_irt` برای `status='confirmed'` و `from_treasury=1`
   - موجودی = ورودی منهای خروجی
   - ledger فقط برای تعدیل دستی استفاده شود، نه اینکه باعث صفر شدن کل صندوق شود.

2. `treasury-ledger.php` را طوری تنظیم می‌کنم که دفتر کل را مستقیم از همان پرداخت‌های واقعی بسازد و اگر query شکست خورد، پیام خطای دقیق JSON برگرداند.

3. در `TreasuryPanel.tsx` یک باکس خطای واضح داخل خود پنل اضافه می‌کنم تا اگر API خطا داد، کاربر دقیقاً ببیند:
   - کدام API خطا داده (`treasury-summary` یا `treasury-ledger`)
   - متن خطا چیست
   - دیگر به‌جای سکوت یا صفر، پیام قابل‌فهم نشان داده شود.

4. برای دیباگ سریع روی هاست، در پاسخ API یک بخش `debug` غیرمخرب اضافه می‌کنم تا مشخص شود API چند رکورد از هر منبع دیده:
   - تعداد پرداخت‌های تاییدشده کاربران
   - جمع ورودی
   - تعداد پرداخت‌های تاییدشده از صندوق
   - جمع خروجی
   - تعداد ردیف‌های ledger

5. فقط همین فایل‌های مرتبط با بانک ترخیصان تغییر می‌کنند؛ دیتابیس و بقیه پنل‌ها دست‌کاری نمی‌شوند.
